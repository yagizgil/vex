# Vex Lexer

Vex Lexer is the stage of the compiler that converts raw source code into a stream of tokens. It is built to be fast, accurate, and easily swappable.

## Implementations

Vex Lexer supports two different implementations:

- **Classic Lexer**: A hand-written, manual lexer. It captures complex rules with custom logic and provides high control.
- **Logos Lexer**: A state-machine based lexer generated by the [logos](https://github.com/maciejhirsz/logos) crate. It is often used for higher performance on larger files.

## Toggle Implementation

By default, the **Classic Lexer** is used. If you want to use the **Logos Lexer**, you must enable the `use-logos` feature.

### Run with Classic Lexer:

```bash
cargo run --release -p vex-cli -- --stats test/all_syntax.vx
```

### Run with Logos Lexer:

```bash
cargo run --release --features use-logos -p vex-cli -- --stats test/all_syntax.vx
```

## Features

- `use-logos`: Enables the Logos-based lexer.
- `inspector`: Supports serialization for the Vex Inspector.
